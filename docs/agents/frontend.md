## Gotchas

- **Results-tab labels must keep `shock applied` vs `contrib to P&L` distinct.** "Shock applied" is the magnitude the LLM proposed for a factor; "contrib to P&L" is `(Σᵢ wᵢ · βᵢ,f) · shock[f]` — the weighted contribution to the total. Never collapse these into a single number.
- **SSE client uses `fetch` + `ReadableStream`, not `EventSource`.** `frontend/src/api.ts::runScenarioStream` reads chunks and splits on `\n\n` frames manually. Reason: `EventSource` is GET-only and doesn't carry POST bodies; the chunked-reader approach also forwards cookies cleanly on same-origin requests. The terminal event is `{"stage":"done","result":...}`; the stream also handles `{"stage":"error","message":...}` by rejecting the promise.
- **Methodology UI is a slide-in drawer, not an inline panel.** `frontend/src/MethodologyDrawer.tsx` renders a right-side drawer over a backdrop with body-scroll-lock + Esc-to-close (`useMethodologyDrawer.ts`). It auto-parses `docs/methodology.md` by splitting on `\n---\n`, slug-ifies section titles, renders an accordion (one expanded section at a time by default, multiple allowed). Deep links from anywhere in the app open the drawer at a specific section: `methodologyDrawer.open("factor-attribution")` matches the first section whose `slug.includes("factor-attribution")`. **The drawer parses markdown sections at render time, so adding a new section to `docs/methodology.md` is enough — no code change needed for the drawer to pick it up.** New deep-link targets DO need a matching `methodologyDrawer.open("<slug>")` call site.
- **Factor names in the Results table are deep-link buttons** (`frontend/src/App.tsx`, `.factor-link` styles). Clicking a factor opens the methodology drawer at `factor-universe`. The `AttributionGuide` component under the attribution toggle deep-links to `factor-attribution`. Both pass through `ResultsPanel`'s new `onOpenMethodology: (section?: string) => void` prop.
- **`refreshAccess` in `App.tsx` is now load-bearing** (Phase 19): it backs the access watcher, the 403→re-check hook, and the session-expired banner via `applyAccess` (see the Phase 19 gotchas below). Don't reintroduce the inline `MethodologyPanel` — the drawer is the canonical surface.
- **`docs/methodology.md` has a "Worked example" heading** inside the Factor-attribution section that the drawer's `parseMethodologySections` flags (via the `hasExample` regex `/worked example/i`) and the CSS highlights with a subtle border accent. It is now a short **qualitative** 2-factor table (the old numeric 3×2 walk-through was deliberately removed — printed magnitudes were a drift magnet; the invariants are pinned synthetically in `tests/test_methodology_worked_example.py`). Keep the heading text when editing, keep the `factor-attribution` section self-contained for the AttributionGuide deep-link, and do NOT reintroduce a numeric example without pinning the shap version.
- **The rail (`<aside className="rail">` with AccessPanel + PortfolioPanel) is conditionally rendered.** `frontend/src/App.tsx` uses `useMediaQuery("(max-width: 1079.98px)")` to choose between inline `<aside>` (desktop) and `<RailDrawer>` (tablet/phone, opened via hamburger). The shared JSX is extracted into a `const railContent` and passed as `children` to RailDrawer. Mounting only one prevents duplicated component instances with diverging local state. The two drawers (rail + methodology) are mutually exclusive — opening one closes the other via `openMethodology` / `openRailDrawer` helpers in App.tsx.
- **`useOverlay` is the shared overlay primitive** ([frontend/src/useOverlay.ts](frontend/src/useOverlay.ts)). It handles `isOpen` state, body-scroll-lock, and Esc-to-close. Critically, the `onClose` callback fires BEFORE `isOpen` flips to false — so consumer-specific cleanup (like clearing `initialSection` on the methodology drawer) runs deterministically for both the explicit `close()` call AND the Esc-key path. `useMethodologyDrawer` is now a thin wrapper around `useOverlay({ onClose: () => setInitialSection(null) })`.
- **`useMediaQuery` is SSR-safe** ([frontend/src/useMediaQuery.ts](frontend/src/useMediaQuery.ts)). Returns `false` during SSR; subscribes to `change` events client-side. Used for the rail-render switch in App.tsx and for the Plotly mobile-layout switch in `ResultsPanel`.
- **Plotly mobile tweaks** ([App.tsx](frontend/src/App.tsx) inside `ResultsPanel`): when `isPhone`, height drops to 320, bottom margin grows to 110, `xaxis.tickangle: -90` with `tickfont.size: 9`. Also `xaxis.automargin: true` as belt-and-braces for clipping. Don't truncate bars — vertical labels handle density.
- **The theme-sensitivity `<table>` in App.tsx is wrapped in `.table-scroll`** so it gets the same horizontal-scroll treatment as the other Results tables. Don't remove the wrapper when refactoring; tests don't catch this regression.
- **Drawer a11y baseline**: both `RailDrawer` and `MethodologyDrawer` set `role="dialog"`, `aria-modal="true"`, and use `useRef` to (a) focus the close button on open, (b) capture `document.activeElement` before open + restore focus to it on close. Full focus-trap (Tab cycling within the dialog) is NOT implemented — that's a follow-up.
- **Methodology citation links use direct external URLs** (DOI / arXiv / publisher pages), NOT anchor links to the bottom References section. react-markdown does not auto-generate heading IDs in the methodology drawer, so `#references`-style fragments wouldn't navigate. If you add a new citation, link to the canonical paper page externally.
- **The `## References` section at the bottom of `docs/methodology.md` is preceded by `---` on its own line.** This is load-bearing: [MethodologyDrawer.tsx::parseMethodologySections](frontend/src/MethodologyDrawer.tsx) splits sections on `\n---\n`. Without the separator, References would fold into the previous section and not be separately navigable in the drawer.
- **Overlay state and overlay shell are separate.** Parent hooks (`useOverlay()`) own body-scroll-lock + Esc-to-close. `OverlayShell` owns backdrop click, `role="dialog" aria-modal="true"`, focus trapping, initial focus, focus return, and the optional close header. Dialog/drawer components should pass their meaningful initial focus ref and render only their body content; do not reinstall local focus-return or focus-trap effects in each overlay.
- **`useFocusTrap(ref, active)` ([frontend/src/useFocusTrap.ts](frontend/src/useFocusTrap.ts)) adds ONLY Tab-cycling containment.** It composes with `useOverlay` (scroll-lock + Esc) through `OverlayShell`; do not duplicate it inside individual drawers/dialogs. It reads `ref.current` inside the handler (capture-phase `document` listener) so a late-mounted panel still traps.
- **`useOverlay` reads `onClose` from a ref.** `close` and the Esc effect depend only on `[isOpen]`; an unmemoized `options` object no longer churns the keydown listener every render. The "onClose fires BEFORE `isOpen` flips false" contract is preserved (and tested).
- **Attribution control is a real radiogroup.** `role="radiogroup"`/`radio` + roving `tabIndex` + Arrow keys; the nav math is the pure `nextEnabledMethod()` in [frontend/src/attributionNav.ts](frontend/src/attributionNav.ts) (skips disabled, wraps) — unit-tested in isolation so it doesn't require rendering `App`.
- **AdjustmentPanel slider range is exactly `[p10, p90]`; the Remove button is the only path to 0.** The valid domain `[p10,p90] ∪ {0}` is disjoint when the envelope is one-signed, which a single continuous slider can't represent. The number input snaps into the valid domain on blur; the slider is disabled when the value sits outside `[p10,p90]` (e.g. a removed factor). Rows with envelope `count < 3` additionally disable BOTH inputs and caption "n=K analog observation(s) — keep or remove" (the Remove button stays live) — mirroring the server-side keep-or-remove rule.
- **"MTM" vs "notional dollar view" — distinct, don't conflate.** *Mark-to-market* = the admin **Shares (MTM)** path that marks share quantities to the as-of close + FX (server-authoritative `result.portfolio_nav`, shown read-only "marked"). *Notional dollar view* = a **client-side NAV knob in `ResultsPanel`** available to **everyone incl. visitors**: dollars are `field × NAV` recomputed **instantly on change — no re-run, no backend call, never cached/keyed**. `charts.ts::parseNav` accepts `$`/commas/`k`/`m`/`b` and rejects non-positive (no silent `NaN`→null). The pre-run NAV field is gone; NAV lives in the results toolbar.
- **Stressed values are client-side and exact** (`charts.ts::buildPositionValuations`): `stressed_value = value + NAV·by_ticker_total[t]`, `stressed_NAV = NAV·(1+total_pnl)`, `Δ% = delta/value` (= the name's scenario return). The Position-valuation table + NAV→stressed metric render **whenever a NAV exists**, independent of the `%/$` toggle (which only flips the waterfall + headline units). Don't duplicate `$ P&L` into the name-level attribution table (it stays pure `%`).
- **Sector/country exposure is a client-side breakdown** from the snapshot's baked `ticker_meta`, served by `GET /api/portfolios/ticker-metadata` (`"Unknown"` for misses so custom books get a complete map). `charts.ts::groupByTag` aggregates `portfolio_holdings` (weight) + `portfolio_pnl.by_ticker_total` (P&L) by tag; the `ExposureBreakdown` component renders it with a Sector/Country toggle. No schema/engine change.
- **(Phase 19) Run/decompose streams are guarded by `frontend/src/runLifecycle.ts`.** `begin()` aborts the prior stream and bumps a sequence; `handleRun`/`handleDecompose` gate progress frames, the final result, catch, AND finally on `isCurrent(seq)` — this closes the stale-stream-overwrites-newer-result race AND stale stepper repaints. `cancel()` aborts without bumping seq so the cancelled run's own finally still resets UI state. Separate lifecycles per action — cancelling a run must not abort a decomposition.
- **(Phase 19) `refreshAccess`/`applyAccess` in `App.tsx` are load-bearing** (replaces the old "intentionally retained" note): `useAccessWatch` polls `/api/access` every 5 min + on visibilitychange (failures swallowed — a network blip must not fabricate a downgrade), every `forbidden`-kind ApiError triggers a re-check, and `applyAccess` raises the session-expired banner ONLY on an un-intentional admin→visitor transition (deliberate lock/unlock passes `intentional: true`).
- **(Phase 19) Two `useOverlay` overlays must never be open simultaneously** — both Esc listeners sit on `window` and would close together. The purge flow therefore CLOSES the ops drawer before opening the type-DELETE-to-confirm `ConfirmDialog`; `openOpsDrawer`/`openMethodology`/`openRailDrawer`/⌘K all close the others first. The toast stack renders at `--z-toast` (120), above the 100/101 backdrop/overlay tier.
- **(Phase 19) Purge must remount BOTH admin panels.** Backend purge deletes scenarios + portfolios + snapshots, but `SavedScenariosPanel`/`PortfolioHistoryPanel` fetch on mount/selection only — App bumps `adminDataEpoch` and passes it as each panel's `key` so deleted records can't linger on screen.
- **(Phase 19) `scrollBehavior()` lives in `frontend/src/motion.ts`** — every JS smooth scroll (MethodologyDrawer sections, run→results auto-scroll, focusUnlock) must import it; the CSS reduced-motion block can't reach JS `scrollIntoView` options.
- **(Phase 19) `TableScroll` (frontend/src/TableScroll.tsx) is the shared scrollable-table wrapper** — `.table-wrap > .table-scroll` with the right-edge fade gated by `useScrollFade` (`has-overflow` class: shown only while hidden content remains to the right, hides at scroll end). Used by TableCard, valuation, decomposition, ExposureBreakdown, AND the OpsDrawer audit log — don't hand-roll the two divs.
- **(Phase 19) CSV exports go through `frontend/src/csv.ts`,** which applies an Excel formula-injection guard to TEXT cells only (`=`, `+`, `-`, `@`, tab, CR get a leading `'`) — number-typed cells stay raw so decimal columns remain machine-parseable. Don't bypass `toCsv` with hand-rolled string joins.
- **(Phase 19) Decompose SSE idle timeout is 120s vs the run stream's 60s** (subset gaps are full pipeline reruns). Both share `api.ts::streamSse`, which owns abort forwarding, idle timing, frame parsing, and the error ladder.
- **(Phase 19) vitest gotcha: never write `beforeEach(() => mock.mockReset())` with an expression-body arrow.** `mockReset()` returns the (callable) mock; vitest registers a function returned from a hook as a TEARDOWN callback and will CALL the mock at cleanup — for a `mockRejectedValue` mock that manufactures an unhandled rejection that fails the test with the mock's own error. Always use a braced body.
- **(Phase 24) The Book profile card renders in the results EMPTY state only** (`BookProfileCard` + `.book-profile-cta` in App.tsx); `bookProfile` state clears on any portfolio-selection/holdings/units change so a stale profile can never describe a different book. Exposure bars are plain token CSS (no Plotly); the fit table uses `TableScroll`; grids carry the `minmax(0, 1fr)` guard.
- **(Phase 27) Pin & compare is pure frontend; result identity is `sameScenarioResult` (charts.ts), NEVER `cache_key`/`reproducibility`** — both are null on valid display paths (adjust → no reproducibility; decompose → neither; saved/permalink → no cache_key). Identity = `scenario_text + market_date + portfolio_key + portfolio_pnl.total_pnl`; the total term makes an adjusted variant read as DIFFERENT so pinned-canonical-vs-adjusted comparisons render. `pinnedEnvelope` in App.tsx is touched by NO other result setter (run/adjust/decompose/saved all go through `setResultEnvelope`), so the pin survives everything until unpin/re-pin.
- **(Phase 27) A comparison shows BOTH sides under ONE attribution method** — `commonAttributionMethod` (charts.ts) picks the lowest map both results carry (explicit → grouped → naive) and the panel labels it; never subtract one methodology's contribution from another's. `ComparisonPanel.tsx` is a self-contained sibling of `.results-stack` (AdjustmentPanel precedent — no `nami-rise` stagger, no `is-stale` dim, the run auto-scroll never targets it); percent-only by design (dollar knobs are per-result, unlike NAVs don't compare); absent-side tickers render "—" with delta-vs-zero semantics (`buildTickerDeltas`).
- **(Phase 28) The three evidence surfaces render as ONE `EvidenceBlock`** (frontend/src/EvidenceBlock.tsx): ±1σ idio band framing, analog replay range, and severity ladder share a CSS range gauge (`buildEvidenceGauge` in charts.ts — null-safe percent positions; old cached payloads render whichever layers exist, NONE → the block renders nothing). The three honesty phrases survive VERBATIM inside one merged caption, each clause gated on its layer ("dispersion floor — not a confidence interval" / "an evidence-base bound, not a joint scenario" / "historical replay, not a forecast") — `EvidenceBlock.test.tsx` pins them. The readout keeps the compact `± x% idio (1σ)` value; the old strips/`AnalogReplayStrip`/`SeverityLadderStrip`/`.idio-band-note` are gone.
- **(Phase 28) `CollapsibleCard` is the drill-layer primitive** (frontend/src/CollapsibleCard.tsx): tier default decided ONCE at mount (desktop ≥1080 opens, compact collapses; `defaultOpen` overrides — theme sensitivity passes an explicit default), user toggles own it afterwards; the body stays MOUNTED but hidden (child state survives) and `.collapsible-body[hidden]{display:none}` is LOAD-BEARING — the class's `display:grid` beats the UA `[hidden]` rule without it. The `summary` prop must carry information ("19 factors shocked · top SPY −30%"), not decoration. **Never wrap the answer layer** (ScenarioReadout, EvidenceBlock, results toolbar, waterfall card). Legacy admin panels are wrapped via `.panel-shell`, whose CSS neutralizes the inner `.result-card` chrome + duplicate heading — don't strip the panels' own markup.
- **(Phase 28) Empty state = one sentence + one `KnowYourBook` card** (tabs: Book profile | Event replay, lazy-fetch on CTA); the 3-step onboarding list and the duplicated seed chips are gone (the scenario card's chips are the ONLY sample selector). Theme sensitivity renders ONLY when `canDecompose` (visitors no longer see a disabled card). `AttributionGuide` is deleted — the `.guide-link` line deep-links the methodology drawer's `factor-attribution` section (single explainer). The ⌘K topbar button is hidden ≤640px (`.cmdk-btn`) — the palette stays a keyboard accelerator and every command keeps a visible control.
- **(Phase 29) `frontend/src/PlotLazy.tsx` is the ONLY Plot import path.** It builds `createPlotlyComponent` (from `react-plotly.js/factory`) over **`plotly.js-finance-dist-min`** — the SMALLEST partial bundle that includes the `waterfall` trace (`basic` does NOT; swapping to a smaller partial silently breaks the attribution chart). It's `React.lazy` so the ~1.2MB plotly chunk loads only when the first result renders; call sites wrap in `<Suspense fallback={<div className="skeleton-block"…>}>`. Tests mock `./PlotLazy` (NOT `react-plotly.js`). The full `plotly.js` stays in node_modules as react-plotly.js's peer but is never imported — `vite.config.ts`'s `chunkSizeWarningLimit: 1300` exists so a full-bundle leak fails loudly at build time. Type shims live in `frontend/src/plotly-modules.d.ts`.
- **(Phase 29) `MethodologyDrawer` is lazy + latch-mounted.** App renders it only after `methodologyMounted` latches true on first open (the react-markdown chunk fetches on demand), then it STAYS mounted so accordion state survives close/reopen. Don't restore a static import (re-bloats the main chunk) and don't unmount on close (loses drawer state).
- **(Phase 29) Identity files live in `frontend/public/`** (favicon.svg + .ico, apple-touch-icon.png, manifest.webmanifest, robots.txt) — Vite copies them to dist root and the existing catch-all serves them. `favicon.svg` is the design source of truth (seigaiha waves, `--bg`/`--accent` literals); the raster icons were generated from the same geometry with a one-shot Pillow script (not committed — regenerate by re-deriving from the SVG). `index.html` carries description/OG/theme-color meta + `viewport-fit=cover`, paired with `env(safe-area-inset-*)` padding in the ≤640px block.
- **(Phase 29) The page `<h1>` is the topbar heading; the rail brand is a styled `<div className="brand-title">`.** Phones hide the rail in a drawer, so the h1 must live in the always-visible topbar. `.topbar h1` pins the former h2 size (`--step-3`) — don't let it inherit the global h1 scale. - **(Phase 29) Run lifecycle is announced via ONE polite live region in App** (`.visually-hidden[role="status"]`), fed by `stageLabel()` exported from RunProgress — the SAME labels the visual stepper shows; completion announces the headline P&L. The active stepper `<li>` carries `aria-current="step"` (pinned in `RunProgress.test.tsx`). Don't add per-component live regions — stacked announcers double-speak.
- **(Phase 29) Friction contracts:** Ctrl/Cmd+Enter in the scenario textarea runs (gated on `runDisabled`, `enterKeyHint="go"`); the passcode input+button is a real `<form onSubmit>` (Enter submits, `autoComplete="current-password"`; `.unlock-row` carries `margin: 0` to kill the UA form margin); `document.title` becomes "{P&L%} · {scenario ≤40ch} — nami" while a result is displayed and resets when cleared; the NAV input selects-all on focus. The footer renders `disclaimers.FOOTER`'s copy verbatim — keep frontend string equal to the backend constant.
- **(Phase 30) The client SSE idle timer re-arms on ANY received bytes** (`api.ts::streamSse` calls `armIdle()` after every `reader.read()`), so keepalive comment frames — which the frame parser deliberately skips (no `data:` line) — keep the stream alive. Don't "simplify" back to rearming only on parsed events: that re-breaks heartbeats. Idle guards stay 60s run / 120s decompose and now fire only on true stalls.
